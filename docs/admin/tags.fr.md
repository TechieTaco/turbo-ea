# Tags

La fonctionnalite **Tags** (**Admin > Metamodele > onglet Tags**) vous permet de creer des etiquettes de classification que les utilisateurs peuvent appliquer aux fiches. Les tags sont organises en **groupes de tags**, chacun avec son propre mode de selection et ses restrictions de type.

## Groupes de tags

Un groupe de tags est une categorie de tags. Par exemple, vous pourriez creer des groupes comme « Domaine Metier », « Cadre de Conformite » ou « Propriete par Equipe ».

### Creation d'un groupe de tags

Cliquez sur **+ Nouveau groupe de tags** et configurez :

| Champ | Description |
|-------|-------------|
| **Nom** | Nom d'affichage du groupe |
| **Mode** | **Selection unique** (un tag par fiche) ou **Selection multiple** (plusieurs tags par fiche) |
| **Restreindre aux types** | Optionnellement limiter quels types de fiches peuvent utiliser ce groupe de tags. Si vide, le groupe est disponible pour tous les types |

### Gestion des tags

Au sein de chaque groupe, vous pouvez ajouter des tags individuels :

1. Cliquez sur **+ Ajouter un tag** dans un groupe de tags
2. Entrez le **nom** du tag
3. Optionnellement definissez une **couleur** pour la distinction visuelle

Les tags apparaissent sur les pages de detail des fiches dans la section **Tags**, ou les utilisateurs peuvent les appliquer ou les retirer.

## Utilisation des tags

- **Detail de la fiche** -- Les utilisateurs voient les groupes de tags disponibles et peuvent appliquer des tags depuis la section Tags
- **Filtrage de l'inventaire** -- La barre laterale de filtres de l'inventaire inclut le filtrage par tags
- **Rapports** -- Les tags peuvent etre utilises comme criteres de regroupement ou de filtrage dans les rapports
- **Portails web** -- La visibilite des tags peut etre activee par portail
